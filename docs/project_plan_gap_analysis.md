# 家装定制行业 AI 转型方案对照清单

对照文件：`家装定制行业AI转型项目方案_V1.0.docx`

## 已落地

- 登录、JWT、角色权限：支持管理员、销售、设计、推广四类演示账号。
- 知识库上传、chunk、embedding、向量检索：支持 txt/md/csv/json/html/docx/pdf/xlsx/xls，本地开源 embedding，ChromaDB 检索，SQL 兜底。
- OpenAI-compatible 聊天接口：`/api/chat/completions` 与 `/v1/chat/completions`。
- DeepSeek 兼容：支持 provider/model/stream 等兼容字段，聊天链路调用真实 DeepSeek；未配置 Key 或上游失败时返回明确错误。
- 企业微信 webhook：支持 URL 验证、回调解析、文本消息回复和群机器人发送。
- 知识库按角色分权：查看、编辑、管理权限可在后台配置。
- Agent 状态控制：暂停/维护状态会阻断对应聊天调用。

## 本轮补齐

- 按方案补齐基础分库：竞品分析库、推广素材库、公共资料库、经营管理库。
- 更新 RAG 检索范围：销售 Agent 增加竞品/公共资料，推广 Agent 增加推广/竞品/案例资料。
- 增加销售客户初筛接口：`POST /api/sales/lead-score`，支持预算、面积、风格、时间、城市、电话和关键词评分。
- 前端增加“销售客户初筛”演示面板，管理员和销售角色可直接试用。
- 增加推广文案生成接口：`POST /api/promo/copy`，使用 DeepSeek + RAG 生成平台文案、标题、标签、CTA 和素材建议。
- 增加设计需求卡接口：`POST /api/design/requirement-card`，将客户自然语言需求结构化为面积、户型、风格、预算、材料偏好、重点空间和设计师待办。
- 增加统一 Agent 分流接口：`POST /api/agent/dispatch`，按用户输入自动路由到销售初筛、设计需求卡、推广文案或普通聊天。
- 前端新增“业务工具”页，集中演示统一分流、推广文案、设计需求卡。

## 仍需继续补

- 推广文案工作流：已能生成文案，但还缺模板管理、历史保存、人工确认发布和多平台排期。
- 设计需求卡工作流：已能结构化解析，但还缺设计师分配、需求卡保存、图片/户型图附件和后续状态流转。
- 竞品分析报告：基于竞品库与外部资料生成对比摘要，当前还没有定时抓取与报告任务。
- 意图识别和 Agent 调度：已有规则分流工具，后续应升级为可配置路由策略、工具调用编排和失败兜底。
- 多平台发布：方案提到小红书/抖音/视频号等，目前只有平台原型数据，没有真实平台 API。
- 视频自动化：方案建议 FFmpeg + AI 脚本/HeyGen，目前尚未实现。
- 设计出图：受酷家乐/三维家 API 限制，当前没有接入生图或 RPA，仅保留后续扩展方向。
- 管理看板：现在是基础指标，缺线索转化率、内容产出、知识库命中率、Agent 成本等真实统计。
- 审计与数据安全：已有操作日志雏形，但缺下载审计、敏感知识库访问审计、密钥轮换和文件存储隔离。

## 建议下一步

1. 先补推广文案生成工具，符合方案“一期切入点”。
2. 再补设计需求卡，能把设计岗位的可行部分跑通。
3. 最后补 Agent 调度层，把销售/设计/推广工具串成统一入口。
